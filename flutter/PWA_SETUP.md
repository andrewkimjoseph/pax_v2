# PWA (Progressive Web App) Setup Guide

This guide explains how to ensure your Flutter web app is installable as a PWA.

## Making the app a PWA

For the app to be installable and work offline as a PWA:

1. **Build with PWA strategy** – Use `flutter build web --release --pwa-strategy offline-first` for every production deploy. This generates `flutter_service_worker.js`. Without it, the service worker is not created and the app will not be installable.
2. **Icons in `web/icons/`** – These four files must exist: `Icon-192.png`, `Icon-512.png`, `Icon-maskable-192.png`, `Icon-maskable-512.png` (192×192 and 512×512; maskable variants should have ~20% safe padding for Android).
3. **HTTPS** – Required for service workers. Firebase Hosting provides this automatically.

**Firebase deploy:** Running `firebase deploy --only hosting` from the `flutter/` directory automatically runs the PWA build via the hosting predeploy script (`scripts/build_web.sh`) and deploys from `build/web`, so no separate build step is needed.

## ✅ What's Already Configured

1. **Web App Manifest** (`web/manifest.json`)
   - App name, icons, display mode, theme colors
   - Icons: 192x192, 512x512 (regular and maskable)

2. **Service Worker Registration** (`web/index.html`)
   - Automatically registers Flutter's service worker after first frame
   - Enables offline caching and installability

3. **Meta Tags** (`web/index.html`)
   - Viewport configuration for mobile
   - Theme color for mobile browsers
   - Apple touch icons for iOS

## 🔧 Build Command for PWA

To build your Flutter web app with PWA support:

```bash
cd flutter
flutter build web --release --pwa-strategy offline-first
```

**Important flags:**
- `--release`: Enables optimizations
- `--pwa-strategy offline-first`: Generates service worker for offline support

**Alternative strategies:**
- `--pwa-strategy offline-first`: Full offline support (recommended)
- `--pwa-strategy none`: No service worker (not recommended for PWA)

## 📋 PWA Installability Requirements Checklist

For a PWA to be installable, it must meet these criteria:

### ✅ Required (All Met)

1. **Web App Manifest**
   - ✅ Present and valid (`web/manifest.json`)
   - ✅ Contains `name` or `short_name`
   - ✅ Contains at least one icon (192x192 or larger)
   - ✅ Has `start_url` defined
   - ✅ Has `display` mode set (we use `standalone`)

2. **Service Worker**
   - ✅ Registered in `index.html`
   - ✅ Generated by Flutter build process
   - ✅ Served over HTTPS (required)

3. **HTTPS**
   - ✅ Required for service workers
   - ✅ Firebase Hosting provides HTTPS automatically

4. **Icons**
   - ✅ 192x192 icon (required minimum)
   - ✅ 512x512 icon (recommended)
   - ✅ Maskable icons for Android

### 📱 Platform-Specific Notes

**Chrome/Edge (Desktop & Mobile):**
- Install button appears in address bar when all requirements are met
- Users can also install via menu → "Install [App Name]"

**Safari (iOS):**
- Uses "Add to Home Screen" feature
- Requires Apple touch icons (already configured)
- Service worker support is limited but improving

**Firefox:**
- Supports PWA installation
- May have different install prompts

## 🧪 Testing PWA Installability

### 1. Build and Deploy

```bash
cd flutter
flutter build web --release --pwa-strategy offline-first
```

### 2. Test Locally (with HTTPS)

For local testing, you can use Firebase Hosting emulator:

```bash
firebase serve --only hosting
```

Or use a local HTTPS server like `http-server` with SSL:

```bash
npx http-server build/web -p 8080 --ssl --cert cert.pem --key key.pem
```

### 3. Verify Manifest

1. Open your deployed app in Chrome
2. Open DevTools (F12)
3. Go to **Application** tab → **Manifest**
4. Check for errors or warnings
5. Verify icons are loading correctly

### 4. Test Service Worker

1. Open DevTools → **Application** tab → **Service Workers**
2. Verify service worker is registered
3. Check status is "activated and running"
4. Test offline mode:
   - Go to **Network** tab → Check "Offline"
   - Reload page - should still work

### 5. Test Installation

**Chrome Desktop:**
- Look for install icon (⊕) in address bar
- Or: Menu (⋮) → "Install Pax"

**Chrome Mobile:**
- Install prompt should appear automatically
- Or: Menu → "Add to Home screen"

**Safari iOS:**
- Share button → "Add to Home Screen"

## 🔍 Troubleshooting

### Service Worker Not Registering

**Check:**
1. App is served over HTTPS (or localhost)
2. Build command included `--pwa-strategy`
3. `flutter_service_worker.js` exists in `build/web/`
4. No console errors in browser DevTools

**Fix:**
- Rebuild with `--pwa-strategy offline-first`
- Clear browser cache and hard reload
- Check browser console for errors

### Install Button Not Appearing

**Check:**
1. All manifest requirements are met
2. Service worker is active
3. App is served over HTTPS
4. User has visited site at least once (engagement requirement)

**Fix:**
- Verify manifest.json is valid (use Chrome DevTools)
- Ensure icons are accessible (check Network tab)
- Wait a few seconds after page load
- Try in incognito mode (to rule out cache issues)

### Icons Not Showing

**Check:**
1. Icon files exist in `web/icons/` directory
2. Icon paths in manifest.json are correct
3. Icons are served with correct MIME type

**Fix:**
- Verify icon files exist: `ls flutter/web/icons/`
- Check manifest.json icon paths
- Ensure icons are PNG format

## 📝 Manifest.json Configuration

Current configuration:

```json
{
    "name": "Pax",
    "short_name": "Pax",
    "start_url": "/",
    "scope": "/",
    "display": "standalone",
    "background_color": "#FFFFFF",
    "theme_color": "#363062",
    "description": "micro tasks rewarded in tokens",
    "orientation": "portrait-primary",
    "prefer_related_applications": false,
    "categories": ["productivity", "finance"]
}
```

**Key fields:**
- `display: "standalone"` - App opens without browser UI
- `theme_color` - Matches your app's deep purple (#363062)
- `background_color` - White background for splash screen
- `start_url` - Where app starts when launched
- `scope` - Defines which URLs the PWA controls

## 🚀 Deployment

When deploying to Firebase Hosting from the `flutter/` directory:

```bash
firebase deploy --only hosting
```

This runs the predeploy script `scripts/build_web.sh`, which executes `flutter build web --release --pwa-strategy offline-first`, then deploys the contents of `build/web`. The service worker (`flutter_service_worker.js`) is generated as part of that build.

**Manual build (e.g. for CI or local testing):**

```bash
./scripts/build_web.sh
# Then deploy, or serve build/web locally
firebase deploy --only hosting
```
